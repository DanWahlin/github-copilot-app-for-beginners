# Contributing

Contributions are welcome when they improve the beginner learning experience.

## What to contribute

Good contributions include:

- clearer beginner explanations,
- fixes to broken links,
- safer screenshots or asset notes,
- troubleshooting improvements,
- corrections to app workflow descriptions,
- sample issue or PR scenario improvements.

## Before you open a PR

1. Keep the course beginner-friendly and practical.
2. Use natural prose.
3. Avoid exact promises about model output.
4. Label optional professional topics as `INTERMEDIATE:` or `ADVANCED:` when appropriate.
5. Do not include secrets, private repositories, customer data, or organization-specific details.
6. If you add screenshots, follow [Screenshot Capture Guide](appendices/screenshot-capture-guide.md) and [Privacy-Safe Course Assets](appendices/privacy-safe-course-assets.md).
7. Check links you changed.

## Pull request checklist

- [ ] The change is scoped to the course material.
- [ ] Beginner readers can follow the wording.
- [ ] Screenshots or examples are privacy-safe.
- [ ] Links point to the right chapter, appendix, or official documentation.
- [ ] No tokens, credentials, private repo names, customer data, or internal details are included.

## Code of conduct

Please follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
